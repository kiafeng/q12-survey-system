# Q12组织氛围诊断调研系统 - 设计规格

## 1. 概念与愿景

一个专业级的企业内部Q12组织氛围诊断调研平台，帮助HR团队高效收集、分析和可视化员工反馈。系统以"清晰、专业、高效"为核心价值，让数据驱动的人力资源决策变得简单直观。

## 2. 设计语言

### 视觉方向
- **风格**: 企业级管理后台风格，专业沉稳但不沉闷
- **参考**: 企业微信管理后台 + 现代化数据看板

### 色彩系统
```
主色(品牌蓝): #1677FF
辅助色: #52C41A (成功/正向指标)
警示色: #FAAD14 (中性/待观察)
危险色: #FF4D4F (负向/需关注)
背景色: #F5F7FA
卡片背景: #FFFFFF
文字主色: #1F1F1F
文字次色: #666666
边框色: #E8E8E8
```

### Q12量表评分配色
```
10分 - 完全符合: #52C41A (绿色)
8-9分 - 符合: #73D13D (浅绿)
6-7分 - 一般: #FAAD14 (黄色)
4-5分 - 不符合: #FF7A45 (橙色)
1-3分 - 极不符合: #FF4D4F (红色)
```

### 字体
- 主字体: "PingFang SC", "Microsoft YaHei", -apple-system, sans-serif
- 数字字体: "DIN Alternate", "Helvetica Neue", monospace
- 标题: 600 weight, 正文: 400 weight

### 空间系统
- 基础单位: 8px
- 卡片内边距: 24px
- 区块间距: 24px
- 圆角: 8px (卡片), 6px (按钮/输入框)

### 动效
- 页面切换: opacity 0→1, 300ms ease-out
- 卡片悬停: transform scale(1.01), box-shadow加深, 200ms
- 数据加载: 骨架屏 + 渐显动画
- 图表: 数据点依次动画入场, 800ms

## 3. 系统架构

### 技术栈
- **前端**: React 18 + Ant Design 5 + ECharts
- **后端**: Node.js + Express
- **数据库**: SQLite (便于部署) / PostgreSQL (生产环境)
- **图表库**: ECharts 5 + @ant-design/charts

### 角色权限
1. **超级管理员**: 全系统权限，可管理所有事业部和部门负责人
2. **事业部管理员**: 仅可查看和管理所辖部门的数据看板
3. **员工**: 仅可填写问卷

## 4. 功能模块

### 4.1 问卷模块
- **问卷内容**: 基于盖洛普Q12设计，包含12道核心问题
- **评分方式**: 1-10分李克特量表
- **问卷分发**: 每个部门/事业部生成独立链接
- **匿名收集**: 员工填写无需登录，确保匿名性
- **进度追踪**: 实时显示各部门回收进度

### 4.2 Q12问卷题目
```
Q1: 我知道公司对我的工作要求
Q2: 我有做好我的工作所需要的材料和设备
Q3: 在工作中，我每天都有机会做我最擅长的事
Q4: 在过去的七天里，我因工作出色而受到表扬
Q5: 我觉得我的主管或同事关心我的个人情况
Q6: 工作单位有人鼓励我的发展
Q7: 在工作中，我觉得我的意见受到重视
Q8: 公司的使命/目标使我觉得我的工作重要
Q9: 我的同事们致力于高质量的工作
Q10: 我在工作单位有一个最要好的朋友
Q11: 在过去的六个月内，有人和我谈及我的进步
Q12: 过去一年里，我有机会学习成长
```

### 4.3 数据看板
- **整体概览**: 全公司Q12平均分、各维度雷达图
- **趋势分析**: 与历史数据的对比(预留)
- **部门对比**: 各事业部/部门Q12得分排名
- **问题预警**: 低于7分的维度红色预警
- **AI分析结论**: 基于数据自动生成诊断报告

### 4.4 AI分析功能
自动分析内容:
- 各维度得分解读
- 优势领域识别
- 改进机会建议
- 与行业基准对比(预留)

### 4.5 后台管理
- 部门/事业部管理(增删改查)
- 问卷链接生成与分发
- 部门负责人权限分配
- 数据导出(Excel格式)
- 问卷回收状态监控

## 5. 页面结构

### 员工端
```
/survey/:token  - 问卷填写页(无需登录)
/thank-you      - 提交成功页
```

### 管理员端
```
/admin/login    - 管理员登录
/admin/dashboard - 数据看板首页
/admin/departments - 部门管理
/admin/surveys     - 问卷管理/链接生成
/admin/users       - 用户/权限管理
/admin/analysis     - AI分析报告
```

## 6. 数据模型

### Department (部门/事业部)
```javascript
{
  id: UUID,
  name: String,           // 部门名称
  code: String,           // 部门编码
  parent_id: UUID|null,  // 上级部门ID(事业部为顶级)
  level: Number,         // 1-事业部, 2-二级部门, 3-三级部门
  survey_token: String,   // 问卷分发Token
  manager_id: UUID,       // 部门负责人ID
  status: Number,        // 1-启用, 0-禁用
  created_at: DateTime
}
```

### User (管理员用户)
```javascript
{
  id: UUID,
  username: String,
  password_hash: String,
  role: Number,          // 1-超级管理员, 2-事业部管理员
  department_id: UUID,   // 关联部门
  created_at: DateTime
}
```

### Survey (问卷记录)
```javascript
{
  id: UUID,
  department_id: UUID,
  answers: JSON,         // {q1: 8, q2: 9, ...}
  ip_hash: String,       // IP哈希(防刷重)
  submitted_at: DateTime
}
```

### Analysis (AI分析报告)
```javascript
{
  id: UUID,
  department_id: UUID,
  report: JSON,          // AI生成的报告内容
  generated_at: DateTime
}
```

## 7. 验收标准

### 功能验收
- [ ] 可创建和管理多个事业部/部门
- [ ] 可按部门生成独立的问卷链接
- [ ] 员工通过链接可匿名提交问卷
- [ ] 后台实时显示各部门的回收进度
- [ ] 数据看板展示Q12各项得分
- [ ] 雷达图可视化展示12个维度
- [ ] AI自动生成诊断分析结论
- [ ] 可给事业部负责人分配权限
- [ ] 事业部管理员仅能看所辖部门数据
- [ ] 数据可导出为Excel

### 性能验收
- [ ] 支持5000份问卷数据
- [ ] 页面加载时间 < 3秒
- [ ] 图表渲染流畅无卡顿

### 安全验收
- [ ] 问卷填写IP去重(同一IP限填1次/部门)
- [ ] Token不可枚举
- [ ] 管理员密码加密存储
- [ ] 事业部管理员数据隔离
